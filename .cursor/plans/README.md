# Plans

**Plan files** are durable, human-readable execution briefs for multi-step work: goal, constraints, file-level steps, and verification. They complement chat context and survive handoffs between sessions or agents.

**Where they live:** `.cursor/plans/**/*.plan.md` (or the same tree under your editor’s plans location if you use a different convention).

Create a new plan when the work spans several files or sessions; link it from issues or PRs when useful. Remove or archive plans when the work is done so the folder stays a map of active intent, not history.
