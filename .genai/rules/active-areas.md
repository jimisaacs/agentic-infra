# Active Areas — Parallel Agent Conflict Visibility

The owner runs 2-6 agents simultaneously. Before modifying files in a major area, check `.cursor/active-areas.md` to see if another agent is already working there. If there's overlap, flag it to the user before proceeding.

When you start substantial work (3+ files in a component), register your area in that file. This is not a lock — it's visibility. The goal is early conflict detection, not prevention.
